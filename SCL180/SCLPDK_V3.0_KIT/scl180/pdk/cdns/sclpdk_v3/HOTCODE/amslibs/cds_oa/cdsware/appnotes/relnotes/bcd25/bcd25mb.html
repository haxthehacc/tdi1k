___________________________________ Date: 06/28/07 Library: bcd25 (bcd25mb) Relevant Specification Documents: * Design Rules: NPB-PS-0793, rev.2 * Electrical Spec: NPB-PS-0784, rev.2 Highlights: * Implemented isolated drain-extended nfets and 40V tolerant diodes * Added 5V native fet and schottky diode. * Parasitic extraction support for calibre and assura * Remember to run trigger callbacks before using this kit. * Limitations are described in the sections below. Software versions tested with this release: * Cadence IC releases IC5.1.41USR3 * Cadence Spectre release MMSIM6.1 (minimum) * Cadence IUS release 5.7QSR2 (AMS Designer) * Synopsys Hspice release 2005.9 (minimum) * ADS release v2006a update1 (minimum) * Cadence Assura 3.1.6USR1_EHF3 (minimum) * Mentor Calibre 2006.1_19 (minimum) - Problems have been discovered in newer versions of the calibre xrc netlist formatter. Process Synopsis: * This bcd25 design kit library supports only the bcd25mb process variant. * The bcd25mb process has 3 layers of aluminum metal. It supports the following: - 5V fets - 5V native fet - 10V drain-extended non-isolated and isolated nfet - 20-40V drain-extended non-isolated and isolated nfet - 8V drain-extended pfet - 20-40V drain-extended pfet - np diffusion diode - n and p drift diffusion diodes - Pwell-Dnw40/Dnw40-Psub diffusion diodes (not 40V equivalents of ddnw/diso diodes) - schottky diode (1 mask add-on) - low value unsilicided, high value unsilicided, silicided, and nwell resistors - 1fF/um2 mim capacitors between metal 2 and metal 3 - vertical npns - vertical pnp - triple well (deep nwell) * This design kit is in the development phase, so issues may arise which will be addressed in future releases. Process Variant Support: * You MUST set the environment variable RDS_CDS_VERIFY_TECH in order to use this design kit. If this variable is not set, the kit will refuse to function. Environment and Kit Updates: * Installation - Review the Jazz Cadence Design Kit Installation Instructions document NPB-PS-0419 for steps to follow in switching between different versions of Design Kits. This document is available on the Jazz Ebizz website. - Updated CIW->Jazz->Application Notes->FAQ with latest design kit version switching instructions. * Flow Changes: - This design kit supports the Cadence inherited connection flow. For more information, refer to the Cadence Inherited Connection Flow Guide in cdsdoc or on sourcelink.cadence.com - The run_cal or Jazz verification flow is not supported in this kit. You will need to migrate to the Calibre Interactive Flow. Documentation can be found in CIW->Jazz->Verification->Calibre Interactive w/xRC. * Kit Updating and other Utilities: - Updated bcd25mb Technology Components web pages. - Trigger Callbacks - Added capability to generate a detailed change report in either dry run or normal operation. Detail report includes details of every parameter value that is changed on any instance in any cellview. - Added warning message if alternate views starting with layout* or schematic* are found when running in hierarchic or library mode. - CIW Jazz menu: New utility JAZZ->Application Notes->Design Kit Environment and Kit Setup Report. This utility generates a report of environment variable and other kit-related information to aid in troubleshooting. - Created new release notes function to show kit or variant specific release notes - Added support for Cadence HTML documentation browser variable CDS_DEFAULT_BROWSER. Default is to use Netscape. - Updated ADE simulation window Jazz Model Library Selection form so it will support PCMT-based corner definitions and allow for model libraries stored outside of the Jazz Design Kit install hierarchy. Use this SKILL command to add additional directories to the model library list in the Model Library Selection form: jazzModelAddDir("/full/path/to/PCMT_Models") The directory provided must contain the simulator directory, for example: /full/path/to/PCMT_models/spectre - Updated Save State functions: - Fixed issue with Jazz saved states and escaped characters. - Updated code to coexist with IC51.41USR3 save state as cellview mode. - Prompt for checkout if Jazz saved state is DM managed. General Kit Limitations: - Artifact Generator. The Artifact Generator may generate shapes that are not DRC correct. If this happens, you can either increase the scale factor or hand-edit the artifact to clean up the errors. Off-grid errors may still occur. Use Snap-To-Grid to correct these violations before cleaning up violations. Device Library Migration: * Trigger callbacks - In the mixed-signal fets, the as/ad/ps/pd values will be updated to reflect calculation changes. This affects devices whose s/d connection is metal2. - The resistor electrical specification values have been changed. - If the "Calculated Parameter" field is set to "Resistance", the width and length will be unchanged. If this field is set to "Length", the width and resistance will be unchanged. When this field is set to "Length", the pcell footprint can change. Device Library Updates: Limitations * bcd25_esdlib library - bigfet_10v_sb device is not lvs clean. This will be fixed in the next update/release. * The rnwell device was not included in the kit. It will be available in a future release. General * Triple well implementation - The triple well is represented in the schematic with the ddnw/diso diodes. The ddnw diode is the dnwRing-Psub diode and the diso diode is the Pwell-dnwRing diode. These two diodes should be used to represent both low voltage (<=8.8V) isolation and high voltage (> 40V) isolation. In the layout, the dnwell layer would be used for low voltage isolation and the dnwell/volt40 layer for high voltage isolation. * Added simulation suport for ADSSim New Devices * nfet5p0_ld10_iso, nfet5p0_ld40_iso: isolated drain-extended nfets - Schematic based devices which contain a *_prim fet and a ddnw diode - There is no layout view for the *_prim device and it is meant to be used only in these schematics. - The ddnw (deep nwell-psub) diode was removed from the model, so this device is now instantiated in the schematic. - maxVds estimate is based on the ldrift length. Only available in devices where the ldrift is scalable. - rdsOn estimate is based on the wf, nf, and ldrift parameters (units are ohms). This is a rough estimate based on connecting to both ends of the s/d metal2 straps. Simulations should be run to find the proper value. The parameter is viewable only when postlayout is set to 0. - postlayout parameter controls the effects of the metal parasitics in the model. When postlayout is set to 0, these parasitics are enabled and when set to 1, they are disabled. - rthFlag parameter controls the thermal resistance effects in the model. When set to 0 the effects are disabled and when set to 1, they are enabled. * nfet5p0_vt0: 5V native nfet * dnp: np diode - schematic contains a 5V ndiode with the deep nwell diode implementation (ddnw/diso diodes) * ddnw40p0/diso40p0 diodes: Dnw40-Psub and Pwell-Dnw40 diodes. - These are diffusion diodes and not not meant to be 40V equivalents of the ddnw/diso deep nwell diodes. * dndrift/dpdrift diodes: n and p drift diodes * dschottky: schottky diode - Note the cathodes need to be manually connected for lvs. Modified Devices * Capacitors - Increased the minimum w/l to 2.92u to meet the via density requirements for the device. This may cause the pcell footprint to change. * High Voltage LD Fets nfet5p0_ld10, nfet5p0_ld40, pfet5p0_ld8, pfet5p0_ld40: drain-extended fets - Schematic based devices which contain a *_prim fet and a ddnw diode - There is no layout view for the *_prim device and it is meant to be used only in these schematics. - The ddnw (deep nwell-psub) diode was removed from the model, so this device is now instantiated in the schematic. - Updated symbol to reflect schematic - Updated the maximum nf and m value to 5000. - Fixed ldrift to snap to 2*grid. ldrift at minimum grid was producing off grid errors in the pcell. This may cause the pcell footprint to change. - Modified maxVds estimate to show that the maximum vds operating voltage should not exceed 40V. Only available in devices where the ldrift is scalable. - rdsOn estimate is based on the wf, nf, and ldrift parameters (units are ohms). This is a rough estimate based on connecting to both ends of the s/d metal2 straps. Simulations should be run to find the proper value. The parameter is viewable only when postlayout is set to 0. - postlayout parameter controls the effects of the metal parasitics in the model. When postlayout is set to 0, these parasitics are enabled and when set to 1, they are disabled. - rthFlag parameter controls the thermal resistance effects in the model. When set to 0 the effects are disabled and when set to 1, they are enabled. - Layout Modifications (This may cause the pcell footprint to change) - Modified the butting diffusion width to match the design rules (N10.13C, N40.13C, P08.13C). - Modified the nwell donut construction to follow the design rules (P08.27C, P40.27C). - Changed the hvpdrift/volt40 to nwell spacing (P40.2) - Optimized the drawing of the guard ring. * Mixed Signal Fets (5V n and p fets) - Increased the maximum nf and m to 100000. - Modified area/perimeter s/d calculations to better handle interconnect style and ablb options. * Resistors - Rs, Rend, and dw values were updated on all resistors. This was done per changes to the electrical specification. This will alter the calculated resistance or length values. The pcell footprint may change. - To avoid changes in the layout, set the Verify parameter to calculate R on all resistors before running trigger callbacks. This will leave the width and length unchanged. - Increased the minimum l to 2u on the rppoly_lo and rppoly_hi devices. This was done per changes to design rules R2/RH2. - Fixed the rppoly_hi to properly draw the tap structure. This may change the pcell footprint.
