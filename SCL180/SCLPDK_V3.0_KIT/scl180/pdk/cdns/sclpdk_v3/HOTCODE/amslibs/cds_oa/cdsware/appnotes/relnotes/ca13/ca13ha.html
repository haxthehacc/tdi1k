___________________________________ Date: 09/17/05 Library: ca13 (ca13ha) Relevant Specification Documents: * Design Rules: NPB-PS-0663, rev.3 * Electrical Spec: NPB-PS-0577, rev.3 Highlights: * Added cells: csmim6, csmimw6_4, cpoly, cpoly3p3, and var_mos3p3 (frontend) * Initial release of lvs, esd_lup, softerc, stress, density, and antenna decks * Included documentation for ca13 on Technology Component Info webpage * Run trigger callbacks before working with this design kit Software versions tested with this release: * Cadence IC releases IC50.33USR2 (minimum) * Mentor Calibre releases 2004.4_11.22 (minimum) Process Synopsis: * This ca13 design kit library supports only the ca13ha process variant. * The ca13ha process has 6 layers of aluminum metal. It supports the following: - 1.2V and 3.3V fets - low value unsalicided and nwell resistors - 3fF/um2 mim capacitors between metal 4 and metal 5 - 6fF/um2 mim capacitors between metal 4, metal 5, and metal 6 - inductors using 3um thick top metal 6 - 3.3V mos varactor - triple well (deep nwell) * This design kit is in the development phase, so issues may arise which will be addressed in future releases. * Currently, only the spectre, spectreS, and ads simulators are supported. Environment Changes: * Review the Jazz Cadence Design Kit Installation Instructions document NPB-PS-0419 for information regarding installing a Design Kit or switching between different versions of Design Kits. This document is available on the Jazz Ebizz website. * Review the CIW->Jazz->Application Notes->FAQ contains steps to perform in the Cadence framework when switching between different versions of Design Kits. * You MUST set the environment variable RDS_CDS_VERIFY_TECH in order to use this design kit. If this variable is not set, the kit will refuse to function. To set this variable, modify your project-specific .cshrc file and change the setting of RDS_CDS_VERIFY_TECH to the process variant you are using. Here are step-by-step instructions: - Go to your project workarea as usual using the source and cdsprj commands. - Type the command "cd ../../../cds_master" - Edit the file cdsPrj.cshrc that is in this directory - Locate the line containing the words "setenv RDS_CDS_VERIFY_TECH" - Change the line to the actual process variant you are using - Save your changes - Close your terminal. You must open a new terminal and start again for the changes to take effect. - source your RDS.cshrc and type the cdsprj command again. * Be sure to run "Trigger Callbacks" before working with this design kit.
