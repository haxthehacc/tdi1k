    What steps should I perform when I switch between design kit releases?

    You have received a new Jazz Design Kit and wish to migrate your existing designs. What steps should you perform?

    Occasionally, the values of resistances, capacitances, or inductances may change between releases of a design kit. Or, the layout Pcells may change slightly between releases of the design kit. These instructions assume you have already completed the new kit installation and set your RDS_ROOT to point to the new kit. We will assume you want to update an entire design library in one step. You will need to repeat BOTH of these steps for every one of your design libraries.

    To update the layout pcells, do the following:

    1. All other users must close their cells. To do this, have all other users quit from the Cadence framework.
    2. Start a Cadence framework session.
    3. Open the Library Manager by choosing the Tools->Library Manager... menu item in the CIW.
    4. In the library manager, go to the library named "softcells"
    5. Delete all of the cells in this library.
    6. As you open the layouts, new softcell cells will be automatically generated for you.
    To trigger the instance callbacks, do the following:
    The resistor/capacitor/inductor update can be performed in one of two modes. In the first mode, the resistor and capacitor value is preserved but the length may change. In the second mode, you can control which parameter (R versus length, or C versus length) will be updated. In all cases, the inductor physical parameters are always preserved.

    Note that if you have flattened any Pcells, the updates will NOT have any effect. You will manually have to edit and re-size the layout shapes for all flattened devices.

    By default, the update will preserve the resistance and capacitance values. The length may change slightly in order to preserve the resistance and capacitance. To trigger all callbacks, preserving R, C, and L values, follow these steps:

    1. All other users must close their cells. To do this, have all other users quit from the Cadence framework.
    2. Start a Cadence framework session.
    3. In the CIW choose the menu item JAZZ->Trigger Callbacks (Lib)s...
    4. Select your design library and OK the form. All views named "schematic" in the design library will have the instance callbacks run. A check-and-save is also performed on the schematics.
    5. Open any existing design layout and choose the menu item JAZZ->Trigger Callbacks->Library.
    6. A form asks you to confirm that you wish to run the callbacks on all layouts.
    7. The instance callbacks are run on all instances of all views named "layout" in the libary. Remember that this can cause device size changes.


    To update devices but preserve the L and W, you only need to update the schematics. Follow these steps instead:

    1. All other users must close their cells. To do this, have all other users quit from the Cadence framework.
    2. Start a Cadence framework session.
    3. Open any existing design schematic and choose the menu item JAZZ->Utilities->Update R and C...
    4. In the new form, set the Scope to Library. Select your design library from the Library cyclic menu.
    5. Set the device type to either resistor or capacitor.
    6. Select the parameter to be calculated. The other parameters will then be preserved. For example, setting Calculate to "R" means that L and W are preserved.
    7. Apply the form.
    8. Select a different device type and repeat the last two steps.
    9. You have now updated all of the resistors and capacitors in your design.
    You must now update the inductors.
    1. Open any existing design schematic and choose the menu item JAZZ->Utilities->Update Inductors...
    2. In the new form, set the scope to Whole Library. Select your design library from the Library cyclic menu.
    3. OK the form. Be sure to check the CIW for any warning or error messages.

    Use of the "Analog" layer during layout

    During LVS the entire background of the layout is considered a single electrically conductive p-well.  If a design uses more than 1 substrate net, use of the Analog layer will be required to pass LVS. The Analog layer is used to chop the substrate and nwells into "islands" of wells on different nets during LVS extraction. Each of these wells has an isolated substrate. However, these islands can be shorted together through taps on the same net (taps connect a well to the related net).

    Notes:

    • Before extraction, the Analog polygons are merged, so butting or overlapping polygons extract as a single substrate net.
    • Devices can only be on one substrate node at a time, so any device that is only partially covered by "Analog" will likely fail extraction.
    • A common source of LVS failure is areas of Analog without any taps, creating a floating net.


    Are standard cells available in the Jazz design kit?
    Characterized standard cells are available for most processes on request as a separate ASIC library. They are not offered as part of this Cadence AMS design kit. Several of the AMS kits contain examples of standard cells which are provided for ease of use in applications where digital characterization is not needed. For example components from this library can be used in an analog design where a single flip-flop is needed and no synthesis or APR (Automated Place & Route) will be required.

    These examples of standard cells are not maintained as interchangable copies of the cells found in the ASIC library. They have views that may not be available in the ASIC library, such as schematic, and they do not support all the views found in the ASIC library. For these reasons, the cells in the AMS design kit cannot be used as a replacement for the cells in the ASIC library. To prevent accidental use in the wrong library in many cases we have prepended "ex_" to the cell names in the AMS design kits. This allows the AMS cells (with "ex_") to be easily distringuished from cells in the ASIC library (without "ex_").


    How to change instances that refer to an unsupported device?

    For example, you are using the sbc18d 1.5 fF MIM cap in your design. This cell (c3t_mim) is no longer shipped with the design kit. How do you change all your instances to use the new device?

    1. Open the top-most schematic of your design hierarchy.
    2. Choose Edit->Search->Replace... in the Schematic window
    3. In the form, change both the Search For and the Replace With cyclic menus to show "cellName".
    4. Set up the rest of the form like so:
            Search For    cellName   ==   c3t_mim
            Replace With  cellName        c3t_mim2
            Search Scope library         (put in your design library)
    5. Click on Apply.
    6. Click on "Replace All".
    7. To keep the original capacitance and W/L ratio:
       Go to the JAZZ menu on the schematic -> Utilities -> Update R and C.
       Choose Device Type=capacitor and Calculate=WL. Select appropriate Scope.
       Click on Apply.

    • When the "New Hierarchy" form appears, go to the bottom and click "Use Template". When the "Use Template" form comes up, choose "Name" to be "spectreVerilog" and click OK.
    •    Change the "view" field from "myView" to "schematic".
    •    In the field of "Stop List", add "functional" to the beginning of the list.
    •    When you press OK,  the Hierarchy Editor window should display all the cells in your schematic with the proper views.  If you only have a symbol and functional view for the verilog block, you should see the "View Found" for
    •    that block as "functional".
    • Save and  exit the Hierarchy Editor.
    4. From the library manager, open the config that you have just created. A form will come up with two options:
    Configuration <cellName> config        yes    no
    Top Cell View <cellName> schematic     yes    no
       Choose "no" for config and "yes" for schematic.
       The schematic will come up.  Notice the banner says "<cellName> config".

    5. Start Analog Artist from the schematic "Tools" pulldown menu. When Artist comes up, look under Design, notice that the "View" is "config".

    6. From Artist,  follow this menu path:  Setup -> Simulator/Directory/Host... In the pop form, in the cyclic field "Simulator", choose "spectreVerilog".

    7. Go back to the schematic/config window.  Check the partition by choosing the menu item Mixed-Signal -> Display Partition

    8. To perform mixed-signal simulations, you have to properly define the
       Interface Element (IE) properties between the analog and digital world.

       TABLE 1.  Interface Element Properties

       Property     Description
       --------     -----------
       a2d_v0       voltage below which to switch to digital state 0
       a2d_v1       voltage above which to switch to digital state 1
       a2d_tx       time duration to switch to x state when voltage is between
                    a2d_v0 and a2d_v1
       d2a_vl       analog equivalent voltage of digital state 0
       d2a_vh       analog equivalent voltage of digital state 1
       d2a_tr       analog rise time when switching from low to high
       d2a_tf       analog fall time when switching from high to low

       In the schematic window, define the IE for the library by choosing the menu item Mixed-Signal -> Interface Elements -> Library...
       When the form comes up, fill in the proper values for the input
       and output of "Model IO".

    9. The Interface Element (IE) properties are already built into the stdcell library components. Set the values of these properties in the Analog Artist simulation session using the menu item JAZZ -> Set IE Vars for spectreVerilog. You can overwrite the default setting of these values.

    10. Setup the simulation as usual, a transient run to a certain time, signals to be saved, etc.

    11. Click on the Netlist and Run button to start the simulation.

    12. How to run spectreVerilog in batch mode:

    • Generate the spectre simulation script (runSimulation) from Analog Artist.

      • You need to run simulation once to create this file.
    • Go to the directory which contains this file:

      • cd your_project_work_area/simulation/cell_name/spectreVerilog/config/netlist
    • Modify the file using your editor vi/text_editor etc.:
    • Remove the following items:

      • env artist_version,
      slvhost host_name
    • Run simulation from this directory as:

      • (a) batch on your workstation: runSimulation
      (b) batch on remote machines using LSF: bsub -q Queue_name "runSimulation"

    What are the contact or via cell purposes in the Design Kit?

    The NTAP cell allows metal1 contact to the nwell using N+ Implant and active (nwell layer not included).
    The PTAP cell allows metal1 contact to the p-substrate using P+ implant and active.
    The M1_D cell allows metal1 contact to active only without n+ or p+ implant's.
    The gcont and cont cells provide metal1 connection to poly1.
    The g12 and v12 cells both provide connection between metal1 and metal2 using the via1 layer.
    The g23, v23, v23m, and v23_mimcap cells provide connection from metal2 to metal3 through the via2 layer.
    The g34 and v34 cells provide connection between metal3 and metal4 using the via3 layer.
    The v45 cell provides a connection from metal5 to metal4 using the via4 layer.

    Does the nfet and pfet pcell style option setting equal to met2(RF) call the special RF fet model? What is the difference between using the nfet or pfet met2 style setting and the met2(RF) setting in the layout?

    The only means of obtaining the special RF model is to instantiate the nfet_rf or pfet_rf pcell. The met2(RF) style selection affects the layout only for multiple slices.

    When multiple slices are desired, the met2(RF) style will generate a layout with separate gate connections for each slice whereas the met2 style option generates a layout with only one gate connection.

    Setup for coarse snapsize and gridsize

    For users / projects who what to use coarse snapsize and gridsize,
    the following variables can be set in .cdsinit or
    /prj/<projName>/cds_master/proj.cdsinit:

    rdsLayUserDefinedSnapSize = 0.25
    rdsLayUserDefinedGridSize = 0.5
    rdsLayUserDefinedMajorGridSize = 5

    Layout window will use those as snapsize and gridsize if they are defined.

    The recommended numbers for different technology are:

    cg18/cg18a/cg18r:
            rdsLayUserDefinedSnapSize = 0.15
            rdsLayUserDefinedGridSize = 0.3
            rdsLayUserDefinedMajorGridSize = 3

    ch2a/ch25r:
            rdsLayUserDefinedSnapSize = 0.2
            rdsLayUserDefinedGridSize = 0.4
            rdsLayUserDefinedMajorGridSize = 4

    bc35/sbc35:
            rdsLayUserDefinedSnapSize = 0.25
            rdsLayUserDefinedGridSize = 0.5
            rdsLayUserDefinedMajorGridSize = 5

    If you see contacts g12, g23 .. in tech library, those are grid-based contacts
    based on the above numbers.

    How to copy design and still use VXL (Virtuoso-XL)?

    Recently you have experienced some hanging in VXL.  The reason is some of your designs are copied from existing designs.   And the schematic has been changed, especially the net names, but in the layout, old connectivity information keeps there.  So when you run VXL, VXL is confused by the differences between schematic and layout, and can't finish the layout examination.  We have no control on the improvement of VXL, so we have to find some workaround: copy layout cell without connectivity information. Unfortunately, we can not remove net information by program.

    When you see untolerable slowness in CELL1 in VXL, do the followings:

    1. kill the process.  (It's very important that you save your changes often,
       otherwise when you kill the process, unsaved changes will be lost)

    2. start cadence again, in library manager, copy CELL1 schematic (ONLY
       schematic) to CELL2 schematic, open CELL2 schematic, run VXL to generate
       all the devices.

    3. Switch to Layout window (Tools -> Layout), if you don't see JAZZ menu,
       enter cltr+E.  JAZZ -> Misc -> Copy Cell...
       In the popup form, select CELL1 layout, and click ok.

       This will copy the routings from CELL1 layout, and move placements
       as they are in CELL1.

    4. Save CELL2, and switch to layout XL (Tools -> Layout XL).
       You will need to move the new pins generated by VXL.

    5. In library manager, copy CELL2 layout back to CELL1 layout.
       You should be able to edit CELL1 in VXL again.

    Naming convention: cell, pin, net

    We have many tools in our design kit. It is important that you name your cells, pins, nets that will conform to all tools.

    1. Always use alphanumeric and _ in names. Do not use non-alpha characters like "+", "-", ".", "#", ...
    2. Do not use net names that are purely a number, like "1". Instead, use a name like "net1" or "in1".
    3. If you are using standard cells, avoid using the global names 'vdd' and 'vss' as the pin names of an individual cell. Using the same pin names for the sub-block pins as those of the current block pin names is OK.
    4. gnd, out, in, vss, vdd, vee, ..., are reserved words for some tools. Never use them in net/pin names.
    5. Some tools are case insensitive and some aren't. Use all lower case (strongly recommeded) or all upper case for all names.
    6. Limit the total number of characters for cell names to 32.

    Cadence Calibre LVS Options

    A UNIX environment variable named "RDS_CDS_RUNCAL_LVS_OPTIONS" is added, it controls the default run_cal options in "JAZZ->Verification->Calibre LVS->Run LVS...". For example, type the following at a UNIX prompt:

    setenv RDS_CDS_RUNCAL_LVS_OPTIONS "-nogates -no_lvs_reduce"

    When running DivaLVS, we always get the message "Warning: Ambiguity points were found and resolved arbitrarily." How should we fix our design to avoid getting these warnings?
    You can ignore the warnings. They are due to the use of the "count" parameter on instances. The Diva LVS tool cannot determine the exact corresponding layout instance to use because the "count" parameter means there are actually multiple layout instances that correspond to the single schematic instance. One of the "count" number of layout instances is chosen and used. You should not see any issue with this. If you wish to resolve the warning, use an iterated instance naming and set count to 1. For example, instead of naming the instance R1 with count of 10, name the instance R1<1:10> with count of 1.
     


    CH25R, CH25A, CG18R, CG18A FAQ
    How to define rule value for hvfets in ch25a?
    To enable the display and editing of rule A, B, C, D of hvfets in ch25a, enter the following in CIW (or add it in .cdsinit):

    rdsCH25AsetGlob("rdsCH25ADefineHvFetRules" t)

    To disable:

    rdsCH25AsetGlob("rdsCH25ADefineHvFetRules" nil)

    By default, they are disabled.

    Can we abut (merge two devices over diffusion) analog and rf fets?
    RF fets (nfet_rf and pfet_rf in ch25r & cg18r), analog fets(nfet, pfet):
       - abutments (merging two devices over diffusion) of RF fets possible
       - abutments of analog fets possible
       - abutments of RF fets with analog fets is not possible

    Can we connect strip resistors in series/parallel?
    Strip resistors (rw3t in ch25a, ch25r, cg18a, cg18r, rnp3t (ch25r, ch25a), rpp3t (cg18a, cg18r) with the strip marking layer (with option VERIFY=RWS) should not be connected in series or parallel. Device reduction in the layout verification LVS will not work correctly in this situation. If series/parallel connection of resistors is necessary, please use the standard resistors without the strip marking layer (with option VERIFY=R option for rnp3t and rph3t, and VERIFY=WL option for rw3t).

    What is the correct usage of mimcaps?
    MIM capacitors (c3t_mim and c3t_mimw in ch25a, ch25r, cg18r, cg18a)

    • Always use the correct device in the schematics.
      • Place "c3t_mim" if you want to use the device over substrate.
      • Place "c3t_mimw" if you want to use the device over well.
    • If you use "c3t_mimw" (either a single instantiation with count>1, or multiple instances of the device), you must name the well pin/net in the schematic, and route the well net and connect it to the appropriate terminal in the layout.

    BC35 FAQ

    Can we overlap the collector contacts of two identical NPN transistors?
    Yes, but, only the L type (low voltage) transistors with a single base finger. All others should have their collectors unmerged. Separately placed, that is.

    Can we place capacitors over resistors or transistors?
    No. Then the verification will fail.

    What is the minimum acceptable spacing between two adjacent resistors? Can we have a 0.025 spacing between the "resblk" rectangles of two adjacent resistors (it passes DRC and LVS)?
    Whatever passes DRC and LVS is fine.

    There is a parameter "tnom" in the MOS model file set to 25. There is a "tref" parameter in the NPN model file set to 22.85. What should we set tnom to in our simulations (BiCMOS) for most accurate results?
    Set tnom=25. The reference temperature is called "tnom" in spectre and the exact equivalent parameter in hspice is called "tref". Both spectre and hspice will evaluate models correctly based on the circuit temperature (temp in spectre and tempdc in hspice) and the reference temperature (tnom in spectre and tref in hspice).


    SBC35 FAQ

    What are the contact or via cell purposes in the Design Kit?
    The NTAP cell allows metal1 contact to the nwell using N+ Implant and active (nwell layer not included). The PTAP cell allows metal1 contact to the p-substrate using P+ implant and active. The M1_D cell allows metal1 contact to active only without n+ or p+ implant's.  The gcont and cont cells provide metal1 connection to poly1. The g12 and v12 cells both provide connection between metal1 and metal2 using the via1 layer. The g23, v23, v23m, and v23_mimcap cells provide connection from metal2 to metal3 through the via2 layer. The g34, v34, and v34qtx cells provide connection between metal3 and metal4 using the via3 layer. The v45 cell provides a connection from metal5 to metal4 using the via4 layer.
     

    What is the difference between gNTAP and NTAP, gPTAP and PTAP, gM1_D and M1_D, gcont and cont, g12 and v12, g23, v23, v23m and v23_mimcap, g34 and v34 and v34qtx?

    The "g" denotes generous, meaning both the layer overlap of contacts (or vias) and contact-to-contact spacing is greater. The following table shows the current values used for each of these:
     
     
    Cell Name cnt layer space lower upper other lower upper
    gNTAP cont 0.6 act met1 cni 0.3 0.3
    NTAP cont 0.4 act met1 cni 0.25 0.15
    gPTAP cont 0.6 act met1 cpi 0.3 0.3
    PTAP cont 0.4 act met1 cpi 0.25 0.15
    gM1_D cont 06 act met1 N/A 0.3 0.3
    1_D cont 0.4 act met1 N/A 0.25 0.15
    gcont cont 0.6 poly1 met1 N/A 0.3 0.3
    cont cont 0.4 poly1 met1 N/A 0.2 0.15
    g12 via 0.55 met1 met2 N/A 0.275 0.275
    v12 via 0.45 met1 met2 N/A 0.2 0.15
    g23 va2 0.55 met2 met3 N/A 0.275 0.275
    v23 va2 0.45 met2 met3 N/A 0.2 0.2
    v23m va2 0.45 met2 met3 N/A 0.4 0.4
    v23_mimcap va2 0.45 met2 met3 N/A 1.1 1.1
    g34 v3a 0.55 met3 met4 N/A 0.275 0.275
    v34 v3a 0.45 met3 met4 N/A 0.2 0.2
    v34qtx v3a 0.5 met3 met4 N/A 0.35 0.4
    v45 via4 5.0 met4 met5 N/A 2 1.5

    Notes:

    • v23m cell is intended for use in the sbc35mx process where met3 ovlp of va2 is 0.4 microns (rule 28.C).
    • v23_mimcap provides greater met3 ovlp of via2 so that the minimum met3 width can be satisfied for use in the sbc35mx process where minimum met3 width is 2.6 microns (rule 28.A).
    • v34qtx cell is intended for use in the sbc35qtx process where met4 ovlp of via3 is 0.4 microns (rule 38.C) and the via3 size is 0.5 micron square versus 0.45 micron square for the sbc35qx process.
    What is the difference between met3 with purpose d3 versus met3 with purpose dg, and similarly between met4 dg, dt, and d4?

    The Jazz SBC35X design kit is written to allow users to design in one of four processes, sbc35mx, qx, qtx, and px. The metal3 and metal4 layers in each of these processes have their own distinct design rule values. Metal3 and 4 widths and spaces, for instance, are different between the four processes. To facilitate layouts of these different metal layer width and space requirements within the same design kit, different layer purpose pairs are used as shown below.

    Process  metal layer purpose  (abbr) metal layer  purpose  (abbr)
    sbc35mx  met3        drawing3 d3     N/A          N/A      N/A
    sbc35qx  met3        drawing  dg     met4         drawing4 d4
    sbc35qtx met3        drawing  dg     met4         drawingt dt
    sbc35px  met3        drawing  dg     met4         drawing  dg

    Note that the user must pick the correct layer purpose pairs listed above to correctly layout and satisfy metal design rule width and space values using guided paths, for example, in their chosen process. During the stream out process, both the met3 dg and d3 layers will be streamed out to the same layer - layer 28. The same holds true for the d4, dt, and dg metal4 purpose pairs - they are all streamed out to the same metal4 layer 38.
