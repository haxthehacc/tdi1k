Known Issues and Limitations

- Migrating a design from PDK v3.1.1 or before will result in LVS errors related to rppoly_hi. This is due to incorrect displayed values in the older PDK
  (simulations were always correct). To fix this use the Trigger Callbacks utility. Use Preview mode to see the changes that will be made, and then turn 
  off Preview mode to commit the updates to your design.

- Oct inductors shift by one grid point due to changes with JIT 5.1
- For Assura only, rule GDD.1.b may give false DRC errors in some cases. Please contact your TowerJazz design support engineer for review before tape-in.
- Rule POL5.10 may give false DRC errors in some cases. Please contact your TowerJazz design support engineer for review before tape-in.
- When placing devices, do not change the name of the device to another, instead please place a new device of the one wanted.  If you do change the name
  then please press the defaults button on the properties window in order to get the correct values (schematic view ONLY).  A solution to this problem is currently being
  worked on. (Updated trigger callbacks in release v5.4 should fix most of these issues.)
- (All CS18 except Q6/Q7) sdMtlWidth will not update correctly for a default placement of tgate fet.  Trigger callbacks needs to be run, or change to a different metal connection style and
  switch back to unconnected.  This will be addressed in a future release.
- Designs that were previously DRC clean may flag the SIA (small isolated active) DRC errors due to change in algorithm. These rules can be waived if using the CS18 variants containing
 "T" (e.g. CS18QT1). Contact your TowerJazz design support engineer for further clarifications.
- var_mos device is currently not supported in CS18Q6*/Q7* variants.
- rrpoly_hi_serp does not currently check spacing, your resistance will also not be off enough to go past the tolerance.  This resistor will be changed to an X device in the next release.
