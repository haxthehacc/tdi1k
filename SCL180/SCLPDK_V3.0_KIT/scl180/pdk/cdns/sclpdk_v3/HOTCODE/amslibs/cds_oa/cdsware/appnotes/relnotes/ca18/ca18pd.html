___________________________________ Date: 05/04/07 Library: ca18 release (ca18pd) Relevant Specification Documents: Design Rules: NPB-PS-0176, rev.13 Electrical Spec: NPB-PS-0173, rev.11 Design Application Note for High Voltage LD Fets: NPB-PS-0578, rev.1 Highlights: * This is an initial development-level kit release for the ca18pd process variant. Issues may arise which will be addressed in future kit releases. * 5 metal 0.18 RFCMOS super-set kit * Process variant and layer setup utility * X-sigma model support * As always, be sure to run "Trigger Callbacks" before working with this design kit. Software versions tested with this release: * Cadence IC releases IC5.1.41USR3 * Cadence Spectre release MMSIM6.0USR2 * Cadence IUS release 5.7QSR2 * Synopsys Hspice release X-2005.9 * ADS release v2005a * Mentor Calibre release 2006.01_19 - Problems have been discovered in newer versions of the calibre xrc netlist formatter. * Cadence Assura releases 3.1.6USR1_EHF3 - Problems discovered in 3.1.6 USR1 caused the move to the 3.1.6USR1 engineering hotfix 3. Process Synopsis: * The ca18pd process has 5 layers of aluminum metal. It supports the following: - 1.8V and 3.3V fets - very low VT 3.3V native fet - high VT 1.8V fets - low value unsalicided, high value unsalicided, salicided poly, and nwell resistors - p+/nwell and mos varactors - 2fF/um2 mim capacitors between metal3/4 and metal4/5 - 4fF/um2 mim stacked capacitor between metal3/4/5 - inductors using 3um thick top metal 5 - deep nwell Process Variant Support: * You MUST set the environment variable RDS_CDS_VERIFY_TECH in order to use this design kit. If this variable is not set, the kit will refuse to function. * Review the Jazz Cadence Design Kit Installation Instructions document NPB-PS-0419 for steps in using a super-set and customizing layers and devices. This utility is found in: CIW->Jazz->Application Notes->Process Variant and Layer Setup * For users of ca18pf, the above document will describe how to go from the ca18pd super-set to a ca18pf like design kit. Kit and Environment: *Installation - Review the Jazz Cadence Design Kit Installation Instructions document NPB-PS-0419 for steps to follow in switching between different versions of Design Kits. This document is available on the Jazz Ebizz website. - Updated CIW->Jazz->Application Notes->FAQ with latest design kit version switching instructions. * Flow Changes - This design kit supports the Cadence inherited connection flow. For more information, refer to the Cadence Inherited Connection Flow Guide in cdsdoc or on sourcelink.cadence.com - The run_cal or Jazz verification flow is not supported in this kit. You will need to migrate to the Calibre Interactive Flow. Documentation can be found in CIW->Jazz->Verification->Calibre Interactive w/xRC * Kit Updating and other utilities: - Updated Technology Components web pages to include ca18pd. - Trigger Callbacks - Added capability to generate a detailed change report in either dry run or normal operation. Detail report includes details of every parameter value that is changed on any instance in any cellview. - Add warning message if alternate views starting with layout* or schematic* are found when running in hierarchic or library mode. - CIW Jazz menu: New utility JAZZ->Application Notes->Design Kit Environment and Kit Setup Report. This utility generates a report of environment variable and other kit-related information to aid in troubleshooting. - Add support for Cadence HTML documentation browser variable CDS_DEFAULT_BROWSER. Default is to use Netscape. - Update ADE simulation window Jazz Model Library Selection form so it will support PCMT-based corner definitions and allows for model libraries stored outside of the Jazz Design Kit install hierarchy. Use this SKILL command to add additional directories to the model library list in the Model Library Selection form: jazzModelAddDir("/full/path/to/PCMT_Models") The directory provided must contain the simulator directory, for example: /full/path/to/PCMT_models/spectre - Update Save State functions: - Fix issue with Jazz saved states and escaped characters - Update code to coexist with IC51.41USR3 save state as cellview mode - Prompt for checkout if Jazz saved state is DM managed General Kit Limitations: - Artifact Generator. The Artifact Generator may generate shapes that are not DRC correct. If this happens, you can either increase the scale factor or hand-edit the artifact to clean up the errors. Off-grid errors may still occur. Use Snap-To-Grid to correct these violations before cleaning up violations. Device Library Updates: General * Added simulation support for hspiceD and AMS in ADE - Updated terminal mapping and operating point information for all devices - spectreS and hspiceS are no longer supported. Please migrate to the spectre and hspiceD simulators respectively. * cascode and tgate fets: Cascode and tgate fets are not included in this release. Better control over FET layout can be achieved using Virtuoso XL abutment and chain/fold functionality. If you use these devices, you will need to migrate to the standard n/p fet devices and Virtuoso XL functionality. Modified Devices * Bipolars (vpnp) - Removed region parameter. Not supported in new simulation model. * Capacitors - Added 4 pin MIM caps over well cmimw2_4, cmimw2_4_m3, and csmimw4_4 - Modified mimcap width/length minimums to pass drc and improve performance. Minimum w/l for cmim2/cmimw2/cmimw2_4 is 2.52u. The minimum value for the the remaining mimcaps is 2.72u. - Added non-physical parameters to support the X-sigma models. Enabling the "Show Sim Parameters" option in the Edit Object Properties form, will display these parameters. - Implemented cmax estimation for poly capacitors (cpoly/cpoly3p3) * Inductors (single and differential square/octagonal) - Implemented JIT4 support - Inductors instantiated from older design kits will need to be replaced in the schematic and layout. Only Inductor Toolbox version 4.0+ is supported. - shield/shield_oct: These cells are the inductor shields. They are not meant to be used directly in your design. Instead, enable the "shield" option in the inductor properties form. * LD Fets - Modified terminals on both devices. nfet3p3_ld: Terminals are: D G S sub, with the sub shorted to the source. pfet3p3_ld: Terminals are: D G S sub, with the nwell shorted to the source. * Mixed-signal fets (1.8/3.3V n and p fets) - Modified area/perimeter s/d calculations to better handle interconnect style and ablb options. - Removed "Integrated" tap option from the native fet (nfet3p3_vt0). This violated rule 52.J. * RF fets (nfet_rf/nfet3p3_rf/pfet_rf/pfet3p3_rf): - Maximum number of fingers was changed to 128. - Updated area/perimeter s/d calculations to better handle interconnect style and ablb options. - Added warning message when s/d connection is set to 1Met or unconnected. The model was extracted using metal2 as the s/d metal. - Modified area/perimeter source and drain fields to always be un-editable. These values are not netlisted and are provided as a reference. * Resistors - Modified rppoly_lo resistor calculation to match new electrical specification values. * Varactors (var_mos) - Increased max fingers to 100 and max slices to 15. - Updated met2_con simulation parameter to be set to 1 when either metal2 connection option is used. Device Library Limitations: * Process Variants: This kit does not hide devices or device parameters when a device is used in an unsupported process variant. Only a warning message is displayed. The DRC decks will show *.Bad violations for unsupported device layers.
