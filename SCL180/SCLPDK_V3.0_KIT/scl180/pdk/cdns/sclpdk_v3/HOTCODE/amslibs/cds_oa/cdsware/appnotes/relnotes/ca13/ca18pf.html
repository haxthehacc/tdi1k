Environment Changes: * You MUST set the environment variable RDS_CDS_VERIFY_TECH in order to use this design kit. If this variable is not set, the kit will refuse to function. To set this variable, modify your project-specific .cshrc file and change the setting of RDS_CDS_VERIFY_TECH to the process variant you are using. Here are step-by-step instructions: - Go to your project workarea as usual using the source and cdsprj commands. - Type the command "cd ../../../cds_master" - Edit the file cdsPrj.cshrc that is in this directory - Locate the line containing the words "setenv RDS_CDS_VERIFY_TECH" - Change the line to the actual process variant you are using - Save your changes - Close your terminal. You must open a new terminal and start again for the changes to take effect. - source your RDS.cshrc and type the cdsprj command again. * Be sure to run "Trigger Callbacks" before working with this design kit. This will allow you to see the latest changes. They were available in older versions of the pdk because they were modeled as bias independent resistors. This calculation was a rough estimation of the drain and source resistance. Currently, they are intrinsic to the core FET model, allowing them to be bias dependent. Also the dc model of the rf fets is now consistent with the mixed-signal model. rd and rs operating point values can still be accessed for the 3.3v rf fets, but these are not useful values and will be removed in the next kit release. * Modified cmax/cmin calcumations on var_mos to improve accuracy. * Added met2_con parameter to var_mos for lvs and simulation. If you are using the default settings of w=5u, l=5u, and area=25p, the l value may appear as 5.005u.
