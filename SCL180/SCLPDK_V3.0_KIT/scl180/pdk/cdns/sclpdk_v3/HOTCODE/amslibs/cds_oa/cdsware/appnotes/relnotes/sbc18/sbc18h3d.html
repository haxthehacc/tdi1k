Verification:
DRC:
* Revision in DNW.21 check.  Originally this rule required that a ptap
  be placed between an nfet gate and nwell if the gate was withing 10um of the
  nwell.  The new approach is to identify the region between the nfet active area
  and nwell (when < 10um).  To pass DRC a ptap of at least 0.25um2 must be within 5um 
  of the identified region.

Verification:
* Adding techfiles directory missing from v3.3 release (CDBA only)
DRC:
* Replaced existing algorithm for checking the density of the first-poly to emitter 
  poly with a more robust algorithm to mainly prevent false errors. In some rare 
  instances, the new algorithm may flag errors that were missed by the previous version
* Rules 68.C and 68.D are intended to prevent situations where you could get leakage
  between n-type regions of the wafer if you don't allow for sufficient pwell implantation
  between them (the NWell2 layer serves to block the PWell). In this case the NWell2
  region is inside the NPN pcell, surrounded by deep trench and a P+ active guard ring.
  In this case there is no risk of any leakage or even parasitic NPN type behavior so you're safe.
  Rules 68.C and 68.D were modified to recognize this condition and not flag it, thus 
  reducing false errors.
* Fixed DRC errors in dpin related to slot contacts
* Added POL (pad over logic) rules
* Revised implementation of rules 2.E and 2.J (ptap spacing to nwell) to flag previous missed 
  errors in some unique instances.
* Added DNW.20 and DNW.21 rules (ptap must be within 10um of nfet inside isolated pwell)
* Modified DRC to flag dual collector NPNs if both collectors are not connected via metal
LVS:
* Modified LVS to fail dual collector NPNs if both collectors are not connected via metal
RCX:
* Modified Assura extraction to correctly copy NPN schematic parameters to extracted instance (OA)

Models:
MOSVAR:
- Added 3.3v MOSVAR (mosvar_3p3) model
NPN:
- Modified "optical" emitter width (EW =0.2um) and short emitter length (EL < 1.1u)
  high-speed NPN models (lo[121][122][232]_hicum) to align with fab-statistics.
  Key change is reduced collector current by 30% and beta from ~700 to 550.
  There is no change to the narrow (W=0.13um) high-speed or to the medium-voltage NPN models.

Process Synopsis:
* This design kit library supports only the sbc18h3 process variant.
* The sbc18h3 process has 6 layers of aluminum metal and supports
  the following:
  - triple well and deep trench isolation,
  - 1.8V and 3.3V FETs,
  - high speed and standard NPNs with deep trench,
  - high gain lateral PNPs,
  - TiN (between metal3/metal4), low and high value unsilicided poly,
    silicided poly, and N-well resistors,
  - junction and MOS varactors,
  - Schottky diodes,
  - 2.8fF/um2 MIM capacitors between metal3/metal4 and metal4/metal5,
  - 5.6fF/um2 stacked MIM capacitors between metal3/metal5,
  - inductors using 2.8um thick metal6 and 2um thick via5.

Process Variant Support:
* The environment variable RDS_CDS_VERIFY_TECH must be set to sbc18h3
  in order to use this design kit.  If this variable is not set, the
  kit will refuse to function.
