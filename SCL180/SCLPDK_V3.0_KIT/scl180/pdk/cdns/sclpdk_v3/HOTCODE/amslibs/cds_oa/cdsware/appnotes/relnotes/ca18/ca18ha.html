Highlights:
* Implemented drain-extended fets and diffusion diodes 
* Included vertical npn and high value unsalicided resistor 
* X-Sigma model support
* Process variant and layer setup utility
* Be sure to run "Trigger Callbacks" before working with this design kit.

Process Synopsis (Development Level):
* The ca18ha process has 6 layers of aluminum metal.  It supports the following:
  - 1.8V and 5V fets
  - 5V native fet
  - 12V drain-extended non-isolated and isolated nfet 
  - 20-40V drain-extended non-isolated and isolated nfet
  - 12V drain-extended pfet
  - 20-40V drain-extended pfet
  - 40V tolerant diffusion diodes 
  - low value unsalicided, high value unsalicided, salicided poly, and 
    nwell resistors
  - p+/nwell and mos varactors
  - 2fF/um2 mim capacitors between metal2/3 and metal3/4
  - 4fF/um2 stacked mim capacitors between metal2/3/4
  - Lateral mimcap structures
  - inductors using 3um thick top metal 6
  - vertical npn
  - triple well
* This design kit is in the development phase, so issues may arise which will be
  addressed in future releases.

Device Library Updates:
General
* Triple well implementation
  - The triple well is represented in the schematic with the ddnw/diso diodes.
    The ddnw diode is the dnwRing-Psub diode and the diso diode is the
    Pwell-dnwRing diode.  These two diodes should be used to represent both low
    voltage isolation and high voltage (> 40V) isolation.  In the
    layout, the dnwell layer would be used for low voltage isolation and
    the dnwell/volt40 layer for high voltage isolation.  Please refer to the
    CA18 Design Rules and CA18HA Design Application Note for guidelines 
    on building these structures. 
* Added simulation support for hspiceD and AMS in ADE
  - Updated terminal mapping and operating point information for all devices
  - spectreS and hspiceS are no longer supported.  Please migrate to the
    spectre and hspiceD simulators respectively.
* Support for Virtuoso-XL auto-abutment.

Limitations
* In certain cases, the integrated taps on the 5V nfets and 1.8V pfets
  may produce drc errors.
  - A workaround for the pfets is to increase the nwell to implant spacing 
  to > 0.43u.  This will be fixed in a future release.
* Abutting the tap rings of the drain-extended fets may produce drc errors.  This will
  be addressed in a future release. 
* 40V pdiodes are not available in this release
  - These have been replaced by the diffusion diodes.
* Process Variants: This kit does not hide devices or device parameters
  when a device is used in an unsupported process variant.  Only a warning
  message is displayed.  The DRC decks will show *.Bad violations for
  unsupported device layers.


New Devices
* cmim2_m3, cmimw2_4_m3: metal3-topmm2-metal4 mimcaps
  - Supported mimcaps:
    cmim2/cmimw2_4: metal2-topmm-metal3
    cmim2_m3/cmimw2_4_m3: metal3-topmm2-metal4
    csmim4/csmimw4_4: metal2-topmm-metal3-topmm2-metal4
* ddnw/diso: New deep nwell implementation using the diso and ddnw diodes.
  Replaces single 3 terminal ddnw diode.  See CIW->Jazz->Tech Components
  web pages for more details.
* ddnw40p0: Dnw40-Psub diode
  - Not meant to be the 40V equivalent of the ddnw deep nwell diode.
* diso40p0_3/dpdrift_3: Pwell-Dnw40 and p-drift diodes
  - Schematic based devices which contain a *_prim diode and a ddnw diode.
    - There is no layout view for the *_prim device and it is meant to
      be used only in these schematics. 
  - diso40p0_3 is not meant to be the 40V equivalent of the diso deep nwell diode.
* dndrift: n-drift diode
* dnp: np diode
  - schematic contains a 5V ndiode with the deep nwell diode implementation
    (ddnw/diso diodes)
* fuse: Renamed fuse_m5rf to fuse
  - fuse_m5rf instances will need to be replaced with fuse.
* ind_oct/ind_diff_oct: Octagonal inductors with JIT4 support.
* nfet5p0_ld12_iso, nfet5p0_ld40_iso: isolated drain-extended nfets
  - Schematic based devices which contain a *_prim fet and a ddnw diode
      - There is no layout view for the *_prim device and it is meant to
      be used only in these schematics.
      - The ddnw (deep nwell-psub) diode was removed from the model, so this
      device is now instantiated in the schematic.
  - maxVds estimate is based on the ldrift length.  Only available in devices
    where the ldrift is scalable.
  - rdsOn estimate is based on the wf, nf, ldrift, and postlayout parameters
    (units are ohms).  This is a rough estimate based on connecting to
    both ends of the s/d metal2 straps.  Simulations should be run to
    find the proper value. 
  - postlayout parameter enables/disables the effects of the metal parasitics in
    the model.  Always disabled in the assura and calibre extracted views (postlayout=1).
  - rthFlag parameter controls the thermal resistance effects in the model.
    When set to 0 the effects are disabled and when set to 1, they are enabled.
* rppoly_hi: High value p+ poly unsalicided resistor.
* vnpn40p0: vertical npn

  - Increased the minimum l to 1.2u for the native fet (nfet5p0_vt0).  This may cause

  - Modified rppoly_lo resistor calculation to match new electrical specification

  - Modified area/perimeter source and drain fields to always be un-editable.

Process Synopsis (Development Level):
* The ca18ha process has 6 layers of aluminum metal.  It supports the following:
  - 1.8V and 5V fets
  - 5V gate oxide drain-extended 20-40V fets
  - 40V pdiodes
  - 5V gate oxide drain-extended 12V fets
  - 5V low VT native fets
  - Lateral mimcap structures
  - low value unsalicided, salicided poly, and nwell resistors
  - p+/nwell and mos varactors
  - 2fF/um2 mim capacitors between metal2 and metal3
  - 4fF/um2 stacked mim capacitors between metal2, metal3, and metal4
  - inductors using 3um thick top metal 6

Process Variant Support
* Review the Jazz Cadence Design Kit Installation Instructions document
  NPB-PS-0419 for steps to follow in switching between different versions of
  Design Kits. This document is available on the Jazz Ebizz website.
* Updated CIW->Jazz->Application Notes->FAQ with latest design kit version
  switching instructions.
* You MUST set the environment variable RDS_CDS_VERIFY_TECH
  in order to use this design kit. If this variable is not set, the kit will
  refuse to function. To set this variable, modify your project-specific
  .cshrc file and change the setting of RDS_CDS_VERIFY_TECH to the process
  variant you are using. Here are step-by-step instructions:
  - Go to your project workarea as usual using the source
    and cdsprj commands.
  - Type the command "cd ../../../cds_master"
  - Edit the file cdsPrj.cshrc that is in this directory
   - Locate the line containing the words "setenv RDS_CDS_VERIFY_TECH"
  - Change the line to the actual process variant you are using
  - Save your changes
  - Close your terminal. You must open a new terminal and start again
    for the changes to take effect.
  - source your RDS.cshrc and type the cdsprj command again.
