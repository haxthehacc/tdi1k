(sbc13ha) Relevant Specification Documents: * Design Rules: NPB-PS-0663, rev.5 * Electrical Spec: NPB-PS-0577, rev.5 * Design Application Note: NPB-PS-0542, rev.5 Highlights: * This is an initial development-level kit release for the sbc13ha process variant. Issues may arise which will be addressed in future kit releases. * Addressed npn contact sizes. Ongoing process development may result in further changes. * Release of all drc/lvs decks for Calibre and Assura. Assura RCX support is also provided. * Be sure to run "Trigger Callbacks" before working with this design kit. * Limitations are described in the sections below. Software versions tested with this release: * Cadence IC releases IC5.1.41 USR3 with IUS5.7 QSR2 * Cadence AMS Designer requires use of IUS5.7 QSR2 * Cadence Spectre release MMSIM6.1 (minimum) * Synopsys Hspice release Z-2007.3-SP1 (minimum) * ADS release v2005a (minimum) * Mentor Calibre 2006.1_19.19 (minimum) - Problems have been discovered in newer versions of the calibre xrc netlist formatter. * Cadence Assura 3.1.7 (minimum) Process Synopsis: * This sbc13 design kit library supports only the sbc13ha process variant. * The sbc13ha process has 6 layers of aluminum metal. It supports the following: - low and medium breakdown npns with peak Ft approximately 200 GHz - 1.2V to 1.5V and 3.3V fets - high threshold 1.2V to 1.5V fets - low and high value unsalicided poly resistors; nwell resistors - 3fF/um2 mim capacitors between metal 4/5 and metal 5/6 - 6fF/um2 mim capacitors between metal 4, metal 5, and metal 6 - square and octagonal inductors using 3um thick top metal 6 - 3.3V mos varactor - vertical pnp - triple well (deep nwell) * This design kit is in the development phase, so issues may arise which will be addressed in future releases. Process Variant Support: * You MUST set the environment variable RDS_CDS_VERIFY_TECH in order to use this design kit. If this variable is not set, the kit will refuse to function. Kit and Environment: * Installation - Review the Jazz Cadence Design Kit Installation Instructions document NPB-PS-0419 for steps to follow in switching between different versions of Design Kits. This document is available on the Jazz Ebizz website. - Updated CIW->Jazz->Application Notes->FAQ with latest design kit version switching instructions. * Flow Changes - This design kit supports the Cadence inherited connection flow. For more information, refer to the Cadence Inherited Connection Flow Guide in cdsdoc or on sourcelink.cadence.com - The run_cal or Jazz verification flow is not supported in this kit. You will need to migrate to the Calibre Interactive Flow. Documentation can be found in CIW->Jazz->Verification->Calibre Interactive w/xRC. * Kit Updating and other Utilities - Updated Technology Library web pages to include sbc13ha. - Trigger Callbacks - Added capability to generate a detailed change report in either dry run or normal operation. Detail report includes details of every parameter value that is changed on any instance in any cellview. - Added warning message if alternate views starting with layout* or schematic* are found when running in hierarchic or library mode. - CIW Jazz menu: New utility JAZZ->Application Notes->Design Kit Environment and Kit Setup Report. This utility generates a report of environment variable and other kit-related information to aid in troubleshooting. - Created new release notes function to show kit or variant specific release notes - Add support for Cadence HTML documentation browser variable CDS_DEFAULT_BROWSER. Default is to use Netscape. - Updated ADE simulation window Jazz Model Library Selection form so it will support PCMT-based corner definitions and allow for model libraries stored outside of the Jazz Design Kit install hierarchy. Use this SKILL command to add additional directories to the model library list in the Model Library Selection form: jazzModelAddDir("/full/path/to/PCMT_Models") The directory provided must contain the simulator directory, for example: /full/path/to/PCMT_models/spectre - Updated Save State functions: - Fixed issue with Jazz saved states and escaped characters. - Updated code to coexist with IC51.41USR3 save state as cellview mode. - Prompt for checkout if Jazz saved state is DM managed. General Kit Limitations: - Artifact Generator. The Artifact Generator may generate shapes that are not DRC clean. If this happens, you can either increase the scale factor or hand-edit the artifact to clean up the errors. Off-grid errors may still occur. Use Snap-To-Grid to correct these violations before hand-editing the artifact to clean it up. Device Library Migration: * Trigger callbacks - In the mixed-signal/rf fets, the as/ad/ps/pd values will be updated to reflect calculation changes. - The low value unsalicided p-poly resistor electrical spec values have changed. - If the "Calculated Parameter" field is set to "Resistance", the width and length will be unchanged. If this field is set to "Length", the width and resistance will be unchanged. When this field is set to "Length", the pcell footprint can change. Device Library Updates: General * Front-end (simulation) support available for spectre, hspiceD, ADSsim, and ams-in-ade. * Layout support available for all devices. * Support for Virtuoso-XL auto-abutment. Limitations * For small wf values, the high vt fets (nfet_vth, pfet_vth) may produce drc errors. - The workaround is to draw a larger HiVTMrk shape which will meet the minimum width and area rules Modified Devices: * Bipolars - Modified the collector and base contact sizes to meet 0.13 drc checks * Resistors (rppoly_lo) - Modified rppoly_lo resistor calculation to match new electrical specification values. Techfile Updates: * Support for Multi-part paths (MPP) - Created MPP templates for: PTAP, NTAP, v12, v23, v34, v45, and v56 - To access MPP templates, manually load the vias_mpp.il file located in $RDS_CDSLIBS/sbc13/techfiles into cadence - Usage guidelines are available on the Technology Library web pages for sbc13.
