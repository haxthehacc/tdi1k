Foundry Switching Capability for cg18a, cj15

We have implemented complete front-to-back foundry switching capability between CONEXANT and TSMC foundries including schematic capture, simulation, device layout generation and verification for the cg18a library. Due to the difference in Espec. values (sheet resistance, end resistance, unit area capacitance, dielectric thickness and constants etc.), properties of resistors, capacitors and inductors will be affected between the foundries.

A. MIM capacitors

There are substantial differences between the Conexant and TSMC processes for the MIM capacitors. The Conexant process (cg18a) supports MIM cap only between M2 and M3. The TSMC 4-layer metal processes (cg18a_tsmc) supports MIM cap only between M3 and M4. Furthermore, no devices are allowed to be placed under the MIM caps and no metal routing is allowed over the MIM caps. This requires creating new components for the MIM caps (symbol, model, layout generators, and layout verification) for the TSMC processes. The new components are called:
  • c3t_mim_tsmc (MIM cap over sub.) for the 4-layer metal process cg18a_tsmc
  • c3t_mimw_tsmc (MIM cap over well) for the 4-layer metal process cg18a_tsmc
The MIM cap devices in the Conexant cg18a process remain unchanged as c3t_mim (MIM cap over sub.), and c3t_mimw (MIM cap over well).

B. Foundry setup

The foundry selection will be based on setting the UNIX environment variable RDS_CDS_FOUNDRY (i.e., setenv RDS_CDS_FOUNDRY "tsmc") prior to running any cadence application (icms, icfb, layout etc.). The foundry selection will be set to the value from RDS_CDS_FOUNDRY environment variable if it is defined. Otherwise an appropriate default value will be set based on the current process (currently by default this would be "tsmc" for cj15, and "cnxt" for any other processes). This variable can be set at project and/or user level. Note that if the value defined by RDS_CDS_FOUNDRY is not a legal choice, then a warning and a beep are issued and the variable is reset to the default for the technology.

D. Device properties display based on foundry selection (resistors/capacitors)

The symbols were modified to add a label which will display an appropriate calculated electrical value depending on the foundry. For example in cg18a, this label toggles between displaying the value of rcalc or rcalc_tsmc for resistors, depending on the foundry. Note that the value is displayed as "rcalc_cnxt=value" for cnxt and "rcalc_tsmc=value" for tsmc. The requested electrical value (rreq) is also displayed as a label such that if there are differences between rreq and rcalc the user can see this visually. Same goes with capacitors, and ccalc and creq.
If a particular device is supported in one foundry but not the other, then rcalc/ccalc label will be set to a value of "**UNSUPPORTED**" as a visual clue that there is a problem. In addition, when user places the device, edit properties on the device, or try and hit the "calculate" button on the device and the foundry does not support it, dialog boxes will appear indicating there is a problem.
Inductor components (l3t, ldiff etc.) based on the Matlab Inductorlib were modified in a similar way. For example in cg18a, the label for the Matlab function name "Process" will change depending on the foundry selection.
We have also implemented the following two schematic RDS pulldown menus that work in sync with the latest changes:
RDS -> Utilities -> Update R and C...

E. Simulation based on foundry selection

The simulation model libraries are based on appropriate foundry specific models. Model files are named in a consistent way for different foundries using the convention: _.lib (Note that the foundryName suffix is not used for the "cnxt" foundry). For example:

libraryNamefoundryNamemodelFileName
cg18acnxtcg18a.lib
cg18atsmccg18a_tsmc.lib
cg18rcnxtcg18r.lib
cj15tsmccj15_tsmc.lib

All available foundry model library selections will be accessible for simulation. User can select the foundry model using the menu:
RDS -> Set Active Library -> Foundry -> "cnxt" or "tsmc"
If the model foundry selection is different from the technology foundry name defined by RDS_CDS_FOUNDRY, a dialog box will appear reminding the user about the mismatch. If user hits OK, simulation will proceed. If user hits Cancel, simulation will be aborted. Note that simulation will fail if there is component mismatch between the model library selections.

F. Layout verification (DRC, LVS, etc.) based on foundry selection

The techfile for layout verification will be automatically chosen based on the foundry selection. For example:

libraryNamefoundryNametechFileName
cg18a cnxt cg18a
cg18atsmccg18a_tsmc

User can choose and select other techfile, if needed, using the menu:
RDS -> Verification -> (Calibre DRC) (Calibre LVS) (Calibre PEX)
-> Process (pick one of the applicable choices)
