___________________________________ Date: 09/09/08 Libraries: sbc18 Relevant Specification Documents: * sbc18qpa Design Application Note NPB-PS-0923 * sbc18 design rules NPB-PS-0179 * sbc18 electrical specification NPB-PS-0267 Highlights: * This is a development level design kit release for the sbc18qpa process variant. Issues may arise which will be addressed in future kit releases. * Superset design kit with front-to-back flow support * Process Variant and Layer Setup * Jazz Inductor Toolbox (JIT) 4.0 support * X-Sigma model capability Software versions tested with this release: * This design kit REQUIRES the Open Access version of Cadence IC software * Cadence IC releases IC6.1.2 ISR 14 (minimum) * Cadence Spectre release MMSIM7.01.053 (minimum) * Mentor Calibre release 2008.1_32 (minimum) - xRC deck was calibrated with this version and may not function properly in older calibre versions. Process Synopsis: * This sbc18 design kit library supports only the sbc18qpa process variant. * The sbc18qpa process has 4 layers of aluminum metal. It contains the following features: - 5.2u thick top metal - 1.8V/5V mosfets, poly capacitors, and diodes - 8V BVCEO npn optimized for PA applications - square and octagonal inductors - 1.8V mos varactor - low value unsilicided, high value unsilicided, silicided, and metal resistors - 2fF/um2 mim capacitor between metal 2 and metal 3 - parasitic vertical pnp - Triple Well; No deep trench support * This process is in the development phase, so future design kit releases may include changes to device behavior. * Currently, only the spectre simulator is supported. Process Variant Support: * You MUST set the environment variable RDS_CDS_VERIFY_TECH in order to use this design kit. If this variable is not set, the kit will refuse to function. Possible values are: sbc18qpa. * Review the Jazz Cadence Design Kit Installation Instructions document NPB-PS-0419 for steps in using a superset kit and customizing layers and devices. This utility is found in: CIW->Jazz->Application Notes->Process Variant and Layer Setup Kit and Environment: * Installation - Review the Jazz Cadence Design Kit Installation Instructions document NPB-PS-0419 for steps to follow in switching between different versions of Design Kits. This document is available on the Jazz Ebizz website. - Updated CIW->Jazz->Application Notes->FAQ with latest design kit version switching instructions. * Flow Changes - This design kit supports the Cadence inherited connection flow. For more information, refer to the Cadence Inherited Connection Flow Guide in cdsdoc or on sourcelink.cadence.com - The run_cal or Jazz verification flow is not supported in this kit. You will need to migrate to the Calibre Interactive Flow. Documentation can be found in CIW->Jazz->Verification->Calibre Interactive w/xRC. * Kit Updating and other Utilities - Include sbc18qpa Technology Components web pages. From the icfb log window, go to Jazz->Technology Components Web. Then select SiGe90 for PA Applications. - Trigger Callbacks - Added capability to generate a detailed change report in either dry run or normal operation. Detail report includes details of every parameter value that is changed on any instance in any cellview. - Added warning message if alternate views starting with layout* or schematic* are found when running in hierarchic or library mode. - CIW Jazz menu - New utility JAZZ->Application Notes->Design Kit Environment and Kit Setup Report. This utility generates a report of environment variable and other kit-related information to aid in troubleshooting. - Added support for Cadence HTML documentation browser variable CDS_DEFAULT_BROWSER. Default is to use Netscape. General Kit Limitations: * Artifact Generator. The Artifact Generator may generate shapes that are not DRC correct. If this happens, you can either increase the scale factor or hand-edit the artifact to clean up the errors. Off-grid errors may still occur. Use Snap-To-Grid to correct these violations before cleaning up violations. General Device Updates: * Included Pcell library and technology file * sbc18_extlib is used to create the calibre extraction view Modified Devices (Incremental changes from previous sbc18 releases): * Inductors (square and octagonal) - Support for Jazz Inductor Toolbox (JIT) 4.0 - Square inductor parameter ranges: 35u<=xsize<=1m 35u<=ysize<=1m 3u<=w<=35u 2.5u<=s<=10u 1.25<=n<=20 (single-ended inductor) 2<=n<=20 (differential inductor) - Octagonal inductor parameter ranges: 35u<=xsize<=1m 35u<=ysize<=1m 3.01u<=w<=35u 2.51u<=s<=10u 1.25<=n<=20 (singled-ended inductor) 1<=n<=20 (differential inductor) - updated inductor search files to follow new parameter ranges * Mixed-Signal Fets (1.8V/5V n and p mosfets) - Added Model Switch field to select PSP or BSIM models for 1.8V mosfets. The 5V mosfets are currently only supported in BSIM. * NPN - Allows multiple emitter width values: 0.2u, 0.6u, 0.9u - Emitter length is scalable from 1.7u<=l<=10.16u. Can be varied up to 40u for the case where ew=0.9u and EBCf=232. This value is valid only in the PADL library where the slotvia has been broken up into shorter segment lengths. - EBCf (emitter/base/collector finger) configurations: 111, 121, 122, 232, 342, 452 - Provided ability to enable thermal resistance effects in the model. - Introduced postlayout simulation option. When enabled, it will include metal parasitics in the pre-layout simulation. For postlayout-simulations, this field will be disabled so that the parasitics extracted by Calibre or Assura can be used. * RF Fets (1.8V/5V rf n and p mosfets) - Added Model Switch field to select PSP or BSIM models for 1.8V rf mosfets. The 5V rf mosfets are currently only supported in BSIM. - Added nqs switch to 1.8V rf mosfets. Provides ability to enable/disable non-quasi-static effects. - Included sa and sb entry fields for WPE for pre-layout simulations. These fields will be seeded with the physical values from LVS extraction for post-layout simulations - Introduced postlayout simulation option. When enabled, it will include metal parasitics in the pre-layout simulation. For postlayout-simulations, this field will be disabled so that the parasitics extracted by Calibre or Assura can be used. - See also Simulation Parameter Update on CDF Property form. Device Library Limitations: * Process Variants: This kit does not hide devices or device parameters when a device is used in an unsupported process variant. Only a warning message is displayed. The DRC decks will show *.Bad violations for unsupported device layers. * Balun and fuse layouts will be available in a future release. They need to be updated to match the new metal 4 design rules. * Removed schottky diode from device library because it required additional process development * An esd library will be delivered in a future release. Technology Library: * Contains layerDefinition and layerRule sections to support the Pcell library and VLE/VXL. * Included vias: v12, v23, v34, v12_g, and v34_g. The *_g vias have larger metal dimensions to pass minimum metal area checks. * Initial support for Multi-part paths (MPP) - Created MPP templates for: PTAP, NTAP, v12, v23, and v34. - Templates can be accessed through the Layout Window->Create Mpp form. - Usage guidelines are available on the Technology Library web pages for sbc18.
